# Keeper Chrome

> [keeper](https://github.com/kimiellen/keeper) 后端的 Chrome 浏览器扩展前端，提供密码管理的侧边栏界面与自动填充能力。

---

## 技术栈

| 技术 | 说明 |
|------|------|
| [WXT](https://wxt.dev) | 现代浏览器扩展框架，支持 HMR |
| Vue 3 + TypeScript | 响应式 UI，全量类型安全 |
| Pinia | 轻量状态管理 |
| Element Plus | UI 组件库 |
| Motion | 动画库 |

运行时环境：Chrome MV3（Manifest V3）

---

## 功能特性

- **侧边栏界面**：点击扩展图标或按 `Alt+.` 打开 / 关闭侧边栏
- **自动填充**：识别当前页面域名，匹配已保存的账号密码并一键填充
- **密码生成**：右键菜单生成高强度随机密码，直接填入输入框
- **凭据保存**：检测到表单提交时提示保存新凭据或更新已有账号
- **数据库管理**：在扩展内完成解锁、创建、切换数据库操作
- **证书固定**：对本地后端的 TLS 证书进行指纹校验，防止中间人攻击

### 使用方法

1. 确保 [keeper 后端](https://github.com/kimiellen/keeper) 已在本机启动（默认 `https://127.0.0.1:8443`）
2. 首次使用时，在浏览器访问 `https://127.0.0.1:8443` 并信任自签名证书
3. 点击扩展图标打开侧边栏，解锁或新建数据库
4. 登录后即可在任意页面使用自动填充与密码生成功能

## 开发

**环境要求**：Node.js 18+、npm

```bash
# 安装依赖
npm install

# 启动开发服务器（热重载）
npm run dev

# 构建生产版本
npm run build

# 打包为 .zip（用于发布）
npm run zip
```

**加载扩展到 Chrome**：

1. 打开 `chrome://extensions`
2. 开启右上角「开发者模式」
3. 点击「加载已解压的扩展程序」
4. 选择 `.output/chrome-mv3` 目录

## 常见问题

**Q：打开侧边栏时提示"请求失败"**  
A：后端未启动，或浏览器尚未信任本地自签名证书。请先访问 `https://127.0.0.1:8443` 并在浏览器中手动信任证书。

**Q：自动填充无响应**  
A：当前页面域名未匹配到任何已保存书签，或数据库处于锁定状态。
